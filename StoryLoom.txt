第零部分：产品概述
0.1 产品愿景与定位
小说编写辅助系统（简称 NAS）旨在成为严肃长篇小说创作者的「数字编辑搭档」——不仅帮助作者「写完」一部长篇，更通过结构化工具、智能校验与知识管理，引导其「写好」一部兼具真实感、叙事张力与文学深度的巨著。
本产品定位为桌面端优先的 SaaS 应用（含 Web 端），以「创作者工作台」形式提供世界观管理、角色系统、情节结构、冲突主题、场景写作、时间线追踪等核心模块，并通过 AI 辅助能力（基于大语言模型）提供智能检测、灵感激发与一致性校验。

0.2 目标用户画像
用户类型	典型特征	核心痛点
严肃业余作者	有固定工作，利用业余时间创作；已有1部以上未完成作品	世界观设定散乱、章节连续性丢失、人物行为前后矛盾
全职网络文学作者	日更3000-8000字；追求读者留存；更新周期长	剧情拖沓、伏笔遗漏、读者催更压力大
出版级职业作家	有责编对接；作品需通过严格审校；追求文学质感	多稿修订版本管理、角色弧光深度不足、主题表达流于说教
写作新手/学习者	初次创作长篇；缺乏系统方法论；容易在中途放弃	不知从何下手、写一段就推倒重来、缺乏结构框架引导

0.3 竞品分析
竞品	优势	劣势	本品差异化
Scrivener	成熟的桌面写作环境，分屏、草稿板功能强大	无 AI 辅助；叙事结构工具偏弱；学习曲线陡峭	提供文学核心要素智能校验 + 中文写作场景深度优化
World Anvil	世界观管理极为细致，社区生态好	情节/冲突管理缺失；与写作流程割裂	一体化创作工作台，世界观直接关联情节卡片与场景
Notion（自建）	灵活；低成本	需要大量手动搭建；无写作专属视图和校验	开箱即用的写作范式，内置文学方法论
ChatGPT/Claude	AI 续写、头脑风暴能力强	无持久化项目状态；容易「幻觉」出不一致情节	AI 作为辅助层嵌入结构化数据，保持一致性

第一部分：非功能性需求
说明：  本部分定义产品全局的质量属性，优先级与功能需求并列，必须在技术方案评审阶段同步评估。

NFR-1 性能需求
场景	指标	目标值
页面首屏加载	LCP（最大内容绘制）	≤ 2.5 秒（P95）
场景卡片保存	保存响应时间	≤ 500 ms
AI 校验功能	从提交到返回结果	≤ 8 秒（P90）；超时显示进度条
关系图渲染	100节点关系图首次渲染	≤ 3 秒
离线模式	核心写作功能（编辑/查看）	完全可用；联网后自动同步

NFR-2 数据安全与隐私
?所有用户创作内容端到端加密存储（AES-256），服务端零明文访问。
?用户对自己的创作内容保有完整版权，平台不得用于模型训练或商业用途（须在服务协议中明确条款）。
?支持用户随时导出全部数据（JSON + 原格式），并可一键注销账号（含数据彻底删除，30日内完成）。
?符合 GDPR、《个人信息保护法》相关要求。

NFR-3 可用性与无障碍
?支持深色模式/浅色模式切换，长时写作减少眼疲劳。
?所有交互控件满足 WCAG 2.1 AA 级可访问性标准。
?关键写作路径（新建场景、查看角色、打开时间线）支持全键盘操作，无需鼠标。

NFR-4 多端与同步
?支持 Web 端（Chrome/Safari/Edge 最新两个大版本）、桌面端（macOS/Windows Electron 包）。
?移动端提供只读查看 + 灵感速记功能（iOS/Android 原生轻量版）。
?多设备实时同步，冲突时以「最后写入」为准，并保留冲突副本供用户手动合并。

NFR-5 AI 能力边界定义
重要：  以下所有「智能/自动」功能必须在技术方案中明确实现路径（规则引擎 / 统计算法 / LLM API 调用），并在 UI 中向用户透明展示 AI 介入的范围与局限性。

功能	推荐实现方式	备注
设定关联度打分（REQ-1.4）	规则引擎：统计设定被引用次数	第一版优先，无需 LLM
节奏心电图（REQ-3.3）	规则算法：基于用户填写的「冲突强度」字段计算	字段须在 REQ-3.1 明确定义
角色声音盲测（REQ-2.6）	LLM 辅助：对台词进行风格嵌入向量对比	需要 API 调用，标注置信度
冲突重叠度分析（REQ-4.1）	规则引擎：对冲突标签进行关联统计	无需 LLM
反派正当性校验（REQ-2.5）	LLM 辅助：依据填写内容给出建议	须添加「仅供参考」免责提示


第二部分：功能需求模块
优先级说明：  P0 = 必须上线（MVP）/ P1 = 核心迭代 / P2 = 完整体验 / P3 = 长期规划

模块一：世界观与背景设定（World-Building & Realism）
【模块定位】 管理故事发生的世界的基础规则，确保设定的自洽性与真实感，遵循「冰山理论」服务于故事，避免沦为无意义的设定展览馆。
【与其他模块联动】 本模块的「设定词条」可被角色档案（模块二）、场景卡片（模块五）和冲突矩阵（模块四）直接引用，修改设定词条时系统自动高亮所有引用处。

用户故事
?作为一名奇幻小说作者，我希望建立一套专属的魔法体系词典，以便在写作时快速查阅规则，避免前后矛盾。
?作为一名历史背景小说的作者，我希望记录关键历史事件时间线，以便场景卡片能自动校验「某角色彼时是否在场」。
?作为一名写了50个以上设定条目的作者，我希望知道哪些设定从未被故事引用，以便决定是否精简或删除。

REQ-1.1 基础物理与社会设定模块
REQ-1.1	基础物理与社会设定模块	P0
功能点	提供分类表单或富文本编辑器，记录「地理风貌」「政治体制」「经济基础」「文化风俗」「阶层划分」五大维度。
支持特性	支持上传参考图片（≤10MB/张）；支持标签化管理；支持全文搜索。
验收标准	1) 用户可在30秒内新建任意分类的设定词条；2) 标签筛选响应时间 ≤ 300ms；3) 图片上传成功率 ≥ 99.5%。

REQ-1.2 核心法则与设定集引擎
REQ-1.2	核心法则与设定集引擎	P0
功能点	建立专属「法则词典」，用于定义魔法体系、科幻黑科技原理或特定行业潜规则，并可设定法则之间的逻辑依赖关系（A法则是B法则的前提）。
自洽性校验	当两条法则存在逻辑冲突时（由规则引擎检测关键词重叠+用户手动标记），系统提示冲突警告，但不强制修改。
验收标准	1) 可创建无限层级的法则嵌套；2) 法则冲突检测在保存时触发；3) 法则词典支持导出为 Markdown。

REQ-1.3 历史编年史与背景事件库
REQ-1.3	历史编年史与背景事件库	P1
功能点	提供历史事件记录表（事件名/时间/影响范围/关联角色）；设立「初始危机/现状」高亮展示区，供作者快速定位故事起点背景。
联动	历史事件可与模块六的宏观纪年表双向同步，避免数据重复录入。
验收标准	1) 历史事件与纪年表条目自动关联，无需二次输入；2) 关联角色字段支持从角色档案中选取。

REQ-1.4 冰山理论设定校验与专业细节库
REQ-1.4	冰山理论设定校验与专业细节库（新增）	P1
功能点	「设定关联度」评分：统计每条设定被场景卡片/角色档案/冲突矩阵引用的次数，生成引用热力图，零引用条目标红提醒。
专业资料库	设立子板块（医学/法律/历史考据/科技等），存储可复用的硬核细节片段，写场景时可直接「引用插入」。
验收标准	1) 引用热力图每日更新；2) 专业资料库支持跨项目共享；3) 零引用设定提醒可被用户忽略/关闭。


模块二：角色档案与关系网（Character System）
【模块定位】 建立全面、立体的角色数据库，强调人物内在逻辑、独特声音以及对立力量的合理性，确保角色不沦为工具人。
【与其他模块联动】 角色的「表层欲望/深层需求矛盾」直接映射到冲突矩阵（模块四）的「人与自我」冲突层；角色的POV设定自动填充场景卡片的叙事视角字段（模块五）。

用户故事
?作为一名作者，我希望系统提醒我角色的「表层欲望」与「深层需求」是否存在矛盾，以便确保角色行为有内在驱动力。
?作为一名多视角叙事的作者，我希望对两个角色的台词进行盲测，以便判断他们的说话风格是否足够有辨识度。
?作为一名作者，我希望可视化地看到「主角 → 导师 → 反派」之间随剧情发展的关系变化，以便规划人物弧光的转折点。

REQ-2.1 角色基础属性与多视角（POV）面板
REQ-2.1	角色基础属性与多视角（POV）面板	P0
功能点	固定字段录入：姓名、年龄、性别、外貌描述、社会身份、所属阵营。支持上传角色参考图（头像）。
视角设定	支持为每个角色指定默认「叙事视角与声音风格」（第一人称亲密感 / 第三人称限制悬念 / 第三人称全知），此设定自动关联场景卡片的POV字段。
快速模式	新建角色时支持「快速模式」（仅填写姓名+核心设定），可事后补充完整信息；「完整模式」开关显示所有字段。
验收标准	1) 快速模式下10秒内完成角色创建；2) 角色图片支持PNG/JPG，自动压缩至 ≤500KB；3) POV设定变更自动同步到已关联场景。

REQ-2.2 角色深层动机矛盾模型（增强）
REQ-2.2	角色深层动机矛盾模型（增强）	P0
功能点	强制要求用户分别填写「表层欲望（想要什么）」与「深层需求（真正需要什么）」，并提供AI辅助校验工具，基于填写内容判断两者是否存在核心矛盾。
联动	「深层需求」字段内容自动同步至冲突矩阵（REQ-4.1）中的「人与自我」冲突类别，供关联使用。
容错设计	此字段为「强烈建议填写」而非「强制阻断」——若留空，系统以黄色警示提醒，但不阻止保存。
验收标准	1) AI校验结果附有置信度百分比；2) 结果可被用户忽略并记录「已知晓」状态；3) 联动至冲突矩阵无需手动操作。

REQ-2.3 人物弧光与缺陷追踪器
REQ-2.3	人物弧光与缺陷追踪器	P1
功能点	记录「致命缺陷/心理创伤」及其在各关键章节节点的状态变化，生成角色弧光曲线（可视化折线图），支持正弧（成长）/负弧（堕落）/平弧（理念坚守）三种模式。
联动	弧光曲线与大纲看板（REQ-3.1）的章节卡片联动，在章节卡片上显示「该章节角色弧光所处阶段」提示标签。
验收标准	1) 弧光曲线节点可拖拽调整顺序；2) 支持打印/导出为PNG；3) 与章节卡片联动无需重复录入。

REQ-2.4 动态角色关系与群像图谱
REQ-2.4	动态角色关系与群像图谱	P1
功能点	可视化网状关系图，支持根据剧情事件动态扩充图谱节点；支持在图谱界面直接手动添加/编辑角色间的连线与关系类型。
关系标签	连线支持类型标签（血缘/恋爱/对立/师徒/同盟）及备注；支持标记「关系转变时间点」（关联到章节/场景）。
配角功能	支持为配角标注其在主线中的「功能角色」（催化剂/镜像/信使等）及对主角的「映射作用」说明。
验收标准	1) 100节点以内图谱流畅交互（≥30fps）；2) 支持按阵营/关系类型筛选显示；3) 图谱数据与角色档案双向同步。

REQ-2.5 对立力量/反派逻辑构建面板（新增）
REQ-2.5	对立力量/反派逻辑构建面板（新增）	P1
功能点	为反派或对立力量提供专属的「正当性建构」模板，要求作者梳理：① 反派的核心世界观（他认为自己的行为是正确的吗？为什么）；② 触发其行动的历史创伤；③ 从反派视角看主角是「威胁」还是「错误」。
AI辅助	LLM 辅助分析填写内容，给出「逻辑一致性评分」及修改建议。结果标注「仅供参考」免责提示。
验收标准	1) 模板三个维度均支持富文本编辑；2) AI辅助评分可关闭；3) 评分结果不影响保存。

REQ-2.6 专属台词本与声音辨识度测试（新增）
REQ-2.6	专属台词本与声音辨识度测试（新增）	P2
功能点	为核心角色建立「语气特征库」（常用句式、高频词汇、表达节奏、禁用词汇），在场景写作时提供实时提示。
盲测工具	从多个角色的台词库中随机抽取片段，去除角色名标记后呈现给作者，测试其是否能正确识别角色——辨识度不足时给出对比提示。
验收标准	1) 盲测结果记录历史准确率；2) 语气特征库支持富文本+标签；3) 实时提示在写作编辑器中以侧边栏形式呈现，不打断写作流。


模块三：核心情节与叙事结构（Plot & Structure）
【模块定位】 提供宏观故事骨架搭建工具，管理主副线交织，控制叙事节奏与张力。
【与其他模块联动】 章节卡片中的「冲突强度」字段驱动 REQ-3.3 节奏心电图；章节卡片关联角色弧光阶段（REQ-2.3）；场景卡片（模块五）从属于章节卡片，形成「大卷→章节→场景」层级。

用户故事
?作为一名作者，我希望用卡片拖拽的方式规划故事的三幕结构，以便直观调整情节顺序而不需要重写大纲文档。
?作为一名连载作者，我希望在每个章节卡片上标注「结尾悬念」，以便提醒自己每章都必须给读者留下继续阅读的动力。
?作为一名作者，我希望可视化地看到「高潮与舒缓」是否均匀分布，以便避免故事中段节奏疲软导致读者流失。

REQ-3.1 动态结构大纲看板
REQ-3.1	动态结构大纲看板（自定义白板/看板模式）	P0
功能点	灵活的可视化看板，支持自定义列（阶段）名称与顺序；内置多题材模板（好莱坞三幕式、英雄之旅、非线性叙事、起承转合）。
卡片特性	支持卡片拖拽与逻辑连线；支持「大卷→章节→场景」层级嵌套展开；每张章节卡片包含：标题、章节摘要、关联角色、所在故事阶段、【冲突强度】（1-5级，必填，用于驱动节奏心电图）、主线/副线标记。
验收标准	1) 拖拽操作响应时间 ≤ 100ms；2) 支持≥500张卡片无明显卡顿；3) 模板切换不覆盖已有内容，仅新增空白列。

REQ-3.2 章节钩子（Hook）与微型悬念生成器（新增）
REQ-3.2	章节钩子与微型悬念生成器（新增）	P1
功能点	在章节卡片中增加【开篇钩子（情感/悬念/行动）】与【结尾悬念（Cliffhanger）】两个填写栏，均为「强烈建议填写」（留空显示黄色提醒，不阻断保存）。
AI辅助	提供「钩子质量评分」：基于填写内容，LLM给出「是否具备让读者翻页的动力」评估，附改进建议。
验收标准	1) 钩子字段在章节卡片中显著可见，不被其他字段遮挡；2) AI评分功能可独立关闭；3) 评分历史可查看趋势变化。

REQ-3.3 主副线交织与节奏（Pacing）心电图（新增）
REQ-3.3	主副线交织与节奏心电图（新增）	P1
功能点	支持对大纲卡片标记「主线（推动冲突）」或「副线（丰富世界/人物）」。系统根据章节卡片的【冲突强度】字段（定义于REQ-3.1，取值1-5，必填）自动生成叙事节奏心电图（折线图），直观展示高潮与缓冲的分布情况。
诊断提示	当连续5章以上冲突强度处于同一水平时，系统主动提示「节奏可能过于平缓/紧张，建议检查」。
数据依赖	?? 本功能依赖REQ-3.1中「冲突强度」字段的录入完整度，字段填写率 < 50%时隐藏心电图并提示补填。
验收标准	1) 心电图实时更新（修改冲突强度后 ≤1秒刷新）；2) 支持按「主线/副线/全部」筛选显示；3) 诊断提示可被单次忽略。


模块四：冲突与主题设计（Conflict & Theme）
【模块定位】 管理故事的多维矛盾点和精神内核，提出深刻叩问。
【与其他模块联动】 「人与自我」类冲突自动从角色的动机矛盾模型（REQ-2.2）拉取数据；核心问题叩问与场景卡片的「场景目标/核心阻碍」字段联动，校验场景是否服务主题。

用户故事
?作为一名作者，我希望在高潮章节同时触发多个层面的冲突，并用可视化方式确认冲突确实「重叠」，以便打造最强有力的叙事张力。
?作为一名作者，我希望用「一个问题」而不是「一句主题」来锚定作品的精神内核，以便在写作中始终保持对主题的追问，而不是直接说教。

REQ-4.1 多维重叠冲突矩阵表（增强）
REQ-4.1	多维重叠冲突矩阵表（增强）	P1
功能点	按四个冲突层级（人与人/权力、人与社会/体制、人与自然/命运、人与自我/道德）记录冲突事件，并与具体章节/场景关联。
冲突重叠分析	系统统计每个场景/章节涉及的冲突层级数量，在大纲看板上标注「多层冲突标签」，并生成「高潮节点冲突密度」报告，辅助作者识别叙事最强点。
数据导入	「人与自我」类冲突条目可从REQ-2.2角色动机矛盾模型自动导入，减少重复录入。
验收标准	1) 冲突条目与章节/场景的关联支持多对多；2) 冲突密度报告支持导出；3) 自动导入后支持手动编辑。

REQ-4.2 核心问题叩问罗盘（增强）
REQ-4.2	核心问题叩问罗盘（增强）	P1
功能点	将传统「一句话主题」升级为「一个核心问题（Core Question）」形式（例如：「繁华的终点是虚无吗？」）。在项目概览页面始终置顶显示，持续提醒作者写作焦点。
反说教校验	在场景卡片中提供校验 Checkbox：「此场景是否通过人物命运自然呈现了对核心问题的叩问（而非直接说教）？」该字段为选填，但系统会在首稿完成后生成「主题显现度」报告。
验收标准	1) 核心问题在所有主界面始终可见（小尺寸水印或固定标题栏）；2) 主题显现度报告在字数达5万时自动触发；3) 反说教Checkbox汇总统计可视化。


模块五：单个场景设计卡片（Scene Template）
【模块定位】 规范微观写作逻辑，打造具有触摸感和真实感的微观单元。
【与其他模块联动】 场景卡片从属于章节卡片（REQ-3.1），POV字段从角色档案自动拉取（REQ-2.1），感官细节提示词可关联专业资料库（REQ-1.4），场景结果变化触发角色弧光追踪器更新（REQ-2.3）。

用户故事
?作为一名作者，我希望在写作某个场景时系统提醒我添加嗅觉或触觉细节，以便避免场景只有视觉描写而缺乏沉浸感。
?作为一名新手作者，我希望系统明确告知我当前场景的叙事视角是否一致，以便避免无意识地从限制视角滑入全知视角。

REQ-5.1 沉浸式场景卡片模板引擎（增强）
REQ-5.1	沉浸式场景卡片模板引擎（增强）	P0
必备区块	① 叙事视角（POV）：下拉选择，来源于角色档案（REQ-2.1），明确视界限制；② 感官细节提示词：勾选并填写视觉/听觉/嗅觉/触觉/味觉锚点（至少勾选2项，否则黄色提醒）；③ 场景目标：角色当下的具体欲望；④ 核心阻碍：阻止目标达成的元素；⑤ 场景结果与情绪变化：成功/失败+触发的情绪状态变化。
视角一致性警告	若同一场景卡片中出现多个POV标记（由作者手动添加分段标记），系统提示「视角切换警告」，作者可确认为「刻意切换」或修改。
联动	场景结果字段中的「情绪状态变化」自动推送到REQ-2.3角色弧光追踪器，作者确认后生成弧光节点。
验收标准	1) 场景卡片与正文编辑器并排显示（分屏）；2) 视角警告在切换时即时触发；3) 弧光推送需作者手动确认，不自动写入。


模块六：时间线与日程管理（Timeline & Pacing）
【模块定位】 精确管理故事的发生时间，避免逻辑漏洞，增强剧情张力。
【数据统一说明（解决模块重叠问题）】 本模块的时间线数据与模块三大纲看板共享同一数据源，通过「故事日期」字段关联。大纲看板侧重叙事结构（章节顺序），时间线视图侧重故事内时间（角色行踪）。两者是同一数据的不同视图，杜绝双重录入。

用户故事
?作为一名多视角叙事的作者，我希望在同一故事日期下看到所有角色各自在哪里、做什么，以便确保不发生角色「瞬移」的逻辑漏洞。
?作为一名悬疑小说作者，我希望追踪某条线索从「埋设」到「揭秘」的全程，以便检查伏笔的回收是否及时且有效。

REQ-6.1 宏观纪年表视图
REQ-6.1	宏观纪年表视图	P1
功能点	提供绝对时间轴标记大背景历史事件；与REQ-1.3历史编年史双向同步，数据来源唯一，视图不同。
验收标准	纪年表与编年史条目修改实时同步，延迟 ≤ 1秒。

REQ-6.2 微观主线倒计时/日程表
REQ-6.2	微观主线倒计时/日程表	P1
功能点	提供基于故事内日历的日程表，支持设置「死线（Deadline）」；死线临近时在大纲看板对应章节卡片上显示倒计时标记，强化叙事张力感知。
验收标准	死线倒计时标记在大纲看板和时间线视图中均可见，两端数据一致。

REQ-6.3 多角色行踪甘特图
REQ-6.3	多角色行踪甘特图/对照表	P1
功能点	直观展示同一故事时间点下各核心角色所处位置和行动；数据来源于场景卡片中的「POV角色 + 故事日期」字段，自动聚合生成，无需额外录入。
验收标准	1) 支持按角色筛选；2) 甘特图中点击条目可直接跳转到对应场景卡片；3) 自动聚合数据无需手动维护。

REQ-6.4 伏笔与信息差追踪系统
REQ-6.4	伏笔与信息差追踪系统	P0
功能点	管理线索的【埋设】【发酵】【回收/揭秘】三个时间节点，分别关联到具体场景/章节；支持标记「信息差」（哪些角色知情，哪些不知情），利用信息不对称制造张力。
写作提醒	若某条伏笔的「埋设」距「回收」超过用户自定义的章节数阈值（默认20章），系统发送写作提醒；若接近全书结尾仍未回收，发出警告。
验收标准	1) 伏笔条目支持状态筛选（仅显示未回收）；2) 提醒阈值可自定义；3) 信息差标记支持按角色分组查看。

第四部分：全局用户体验设计原则
4.1 分层录入（快速模式 vs 精细模式）
针对「表单填写负担过重」的核心风险，所有包含5个以上字段的模块（角色档案、场景卡片等）均须实现分层录入：
?「快速模式」：仅展示2-3个最核心字段，支持在灵感爆发期快速录入。
?「精细模式」：展示全部字段，适合创作间隙或回顾时补充细节。
?系统记忆用户上次使用的模式，默认沿用。

4.2 「强烈建议」而非「强制阻断」
v2.0 中部分「强制填写」设计会打断创作心流。v3.0 全面调整原则：
原设计	v3.0 调整	适用场景
强制填写（阻断保存）	黄色警示提醒（不阻断）	钩子/悬念、动机矛盾、感官细节
强制选择	默认值+可修改	叙事视角、冲突类型
系统自动操作	推送给用户确认	弧光节点更新、冲突数据导入

4.3 模块联动可视化
所有跨模块的数据联动须在 UI 上清晰标注数据来源，例如：场景卡片的「POV」字段显示「来自角色档案 · 林尽染」，并提供「跳转查看」链接。这有助于用户理解系统整体逻辑，减少数据孤岛感。

4.4 写作编辑器优先原则
所有辅助面板（角色信息、场景卡片、伏笔追踪）均以「侧边栏折叠」形式呈现，确保正文编辑区域始终是界面的视觉主体。用户可一键切换为「专注写作模式」（隐藏所有面板，仅保留正文编辑器）。
